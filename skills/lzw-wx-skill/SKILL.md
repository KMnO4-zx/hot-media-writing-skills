# 量子位 · 写作操作系统

> 追踪人工智能新趋势，关注科技行业新突破。

## 核心定位

量子位（QbitAI）是中国AI领域最活跃的科技自媒体之一，面向AI从业者、开发者和科技爱好者。其写作风格的核心特征：

- **口语化但有料**：像朋友发微信一样讲技术，但信息密度不低
- **玩梗但不低俗**：敢用doge、网络热梗、emoji，但不影响专业判断
- **第一人称叙事**：作者有个性、有态度、有情绪，不是冷冰冰的机构号
- **节奏感极强**：短句为王，一句一段，制造阅读快感

## 文章类型模板（5种）

### 类型1: 热点快讯（Breaking + 评论）

**标题公式**：人名/公司 + 核心事件 + 数字/影响（感叹号常用）

**示例**：
- 「马斯克人在北京，xAI发布首款Coding Agent」
- 「林俊旸果然创业了！一个"Qwen负责人"头衔值135亿」
- 「田渊栋AI创业估值315亿，老黄苏妈都投了」

**结构**：
1. 一句话定调（口语化，制造悬念）
2. 核心事件展开（快节奏信息输出）
3. 产品/技术细节（如有）
4. 人物背景深挖（故事性强）
5. 行业影响 + One More Thing彩蛋
6. 互动收尾

**篇幅**：1500-3000字
**节奏**：极快，段落平均2-3句，大量一句成段

---

### 类型2: 产品体验/测评（第一人称叙事）

**标题公式**：强动词/口语钩子 + 产品核心卖点

**示例**：
- 「重生之我在AI时代当老板：让一群Agent互相PUA」
- 「8岁小学生idea直接变应用，秒哒3.0刚刚把AI应用门槛打没了」

**结构**：
1. **个人体验切入**（"我做了XX"/"刚刚XX发布了"）
2. **使用过程叙事**（第一人称，带情绪反应）
3. **功能拆解**（边体验边讲原理）
4. **效果展示**（截图+评价）
5. **总结判断**（值不值、好不好用）

**篇幅**：2000-4000字
**节奏**：叙事驱动，像在讲故事，穿插吐槽和惊叹

---

### 类型3: 技术解读/论文速递

**标题公式**：反常识钩子/疑问 + 技术核心贡献

**示例**：
- 「挑战扩散自回归统治！字节提出视觉生成第三种路线」
- 「160行代码读懂LeCun的JEPA世界模型」
- 「当SFT遇上RL：基于样本学习阶段的动态策略优化机制」

**结构**：
1. **口语化导入**（"ber！"/"说实话"/"u1s1"）
2. **类比说明**（把技术比作日常事物）
3. **技术原理**（用人话讲，偶尔穿插原始术语）
4. **实测/Demo展示**
5. **论文/代码链接**

**篇幅**：1500-3000字
**节奏**：轻松→深入→轻松，技术段落前后必有口语缓冲

---

### 类型4: 行业事件/八卦（故事驱动）

**标题公式**：戏剧性动词 + 核心冲突/反转

**示例**：
- 「Done！硅谷分拣快递的人类工作，没了」
- 「妙鸭相机解散！9.9元AI初代网红应用落幕」
- 「英伟达给黄仁勋儿女涨薪了！年薪百万美元」

**结构**：
1. **吸睛事实**（最戏剧性的点先抛出）
2. **事件详情**（叙事展开，有起承转合）
3. **背景/历史**（人物故事、公司发展脉络）
4. **业内反应**（网友评论、社区讨论）
5. **One More Thing**（额外彩蛋或后续预告）

**篇幅**：1500-3000字
**节奏**：故事驱动，有高潮有转折，允许八卦语气

---

### 类型5: 深度产业报道（软文/合作向）

**标题公式**：行业趋势判断 + 公司/产品定位

**示例**：
- 「手机的智能体AI，正在因为天玑全面跃升」
- 「人手一个数据库，Kimi背后这套AI基建到底有多能扛？」

**结构**：
1. **行业趋势铺垫**（数据+判断）
2. **痛点分析**（三层递进展开问题）
3. **解决方案呈现**（产品/技术如何解决）
4. **生态/合作伙伴**
5. **展望未来**

**篇幅**：3000-5000字
**节奏**：比其他类型正式，但仍保持量子位口语底色

## 表达DNA

### 标题规则

| 技巧 | 公式 | 示例 |
|------|------|------|
| 感叹号冲击 | 事件 + ！ | 「妙鸭相机解散！」 |
| 数字前置 | 金额/估值/时间 | 「估值315亿」「8小时不间断」 |
| 口语钩子 | Done！/果然/竟然 | 「林俊旸果然创业了！」 |
| 人名引流 | 名人 + 动作 | 「马斯克人在北京」 |
| 反问/设问 | XX？ | 「气象预测长期误差太难搞？」 |
| 网感标题 | 重生之/Done！ | 「重生之我在AI时代当老板」 |
| 引号强调 | "XX" | 「"Qwen负责人"头衔值135亿」 |

### 开头模式

**模式A - 一句定调（最常用）**：
> 马斯克人在中国，大半夜的还没忘在x上给自家xAI站台。

**模式B - 口语感叹**：
> 终于，不用一直对AI说「继续」了……

**模式C - 事实冲击**：
> 离职Meta后的田渊栋，现身独角兽联创名单，成了AI创业合伙人。

**模式D - 反问/设问**：
> ber！这个五一假期，我也是真够忙的：

**模式E - 网络梗**：
> 不er，我左键右键点了20多年，你现在告诉我鼠标开始「脑子」了？？

### 句式偏好

- **平均句长**：10-20字为主（极短！比机器之心短很多）
- **一句成段**：大量使用，制造阅读节奏
- **段落长度**：1-3句为主，超过5句罕见
- **感叹/反问比例**：约20-25%（远高于机器之心）
- **第一人称**：高频使用「我」，作者是有血有肉的人
- **括号吐槽**：*（doge）*、*（bushi）*、*（咳咳）* 常见

### 核心口语句式

| 功能 | 句式 |
|------|------|
| 引出重点 | 「没错，XX」「划重点！」 |
| 情绪反应 | 「不是？？」「属实XX了」「真就XX」 |
| 简化表达 | 「说白了就是XX」「一句话总结就是：」 |
| 转场 | 「但u1s1，XX」「说实在话，XX」 |
| 递进 | 「更别说XX」「甚至XX」「这还没完」 |
| 调侃 | 「也是很「拟人」了（doge）」 |
| 收束 | 「就看XX接下来能搞出什么了」 |
| 设问 | 「那么问题来了：」「所以说XX」 |

### 词汇特征

**高频口语词（量子位标志性）**：
- u1s1（有一说一）、ber、属实、整活、搞出、端出
- 摇人、站台、下场、冲击、拉满、打没了
- 卷到飞起、轻车熟路、一口气、一把梭

**情绪标记词**：
- 正面：炸裂、逆天、离谱、绝了、太对味了
- 惊讶：不是？？、居然、竟然、好家伙
- 调侃：笑了、打工人笑了、这波XX、懂的都懂
- 中性：说实在话、坦白讲、仁者见仁

**技术混搭词**：
- 赛道、路线、架构、范式（正式）
- 但穿插在口语中使用，不显突兀

**禁忌词**：
- 不用「笔者」「我们认为」等书面腔
- 不用过于学术的连接词（「综上所述」「由此可见」）
- 不无根据最高级但可以用带调侃的最高级

### 标点符号习惯

- **！！**：允许连用（但不超过两个），表达真实惊讶
- **？？**：连用表达困惑/震惊「不是？？」
- **……**：用于制造悬念或语气停顿
- ****加粗****：用于关键产品名、核心概念、金句
- **「」**：技术概念和产品功能
- *（斜体括号）*：作者内心吐槽，如 *（doge）*、*（bushi）*
- **——**：较少使用，量子位偏好短句拆分而非破折号补充

### 英中混用规则

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| 产品名 | 保留英文 | Grok Build、Claude Code、Cursor |
| 网络用语 | 保留英文 | One More Thing、Demo、vibe coding |
| 技术术语 | 英文为主 | Agent、NPU、Scaling Law、RL |
| 人名 | 中文名优先，无中文名用英文 | 黄仁勋、马斯克、Richard Socher |
| 社交平台 | 英文 | X（推特）、HuggingFace |

## 结构元素

### 数据呈现方式

- 数字直接加粗：**6.5亿美元**、**300刀/月**
- 换算为人民币：46.5亿美元（约316亿元人民币）
- 对比增强：「从每日1.2亿次到8.7亿次，7倍增长」
- 不用复杂表格，偏好直接在正文中用加粗数字

### 引用模式

- **人物原话**：用引用块 `>` 格式
  > 我们将尽可能保持团队小而精，最终把很多事情委托给我们的Agent。
- **网友评论**：直接引用+截图
- **社交媒体发言**：「XX在X上表示：」

量子位的类比更加日常化、年轻化：

| 技术概念 | 类比方式 |
|---------|---------|
| 扩散模型画法 | 「像个笨拙的学生，无论是画苹果还是巴洛克壁画，都要一笔一笔画」 |
| 自回归模型缺陷 | 「粗心大意没有橡皮，前面画错了还会越画越离谱」 |

- 优先用年轻人能秒懂的比喻
- 允许用游戏/动漫/网络文化类比
- 类比可以带幽默感，不需要严肃精确

## 写作工作流

### Step 1: 确定文章类型和语气

根据素材判断5种类型，选择对应模板。注意：
- 热点快讯 → 最口语、最快节奏
- 产品体验 → 第一人称叙事
- 技术解读 → 口语+技术混搭
- 行业八卦 → 故事+调侃
- 产业报道 → 相对正式但仍有量子位味道

### Step 2: 构建标题

确保：
- 有信息增量（读者看标题知道发生了什么）
- 有情绪钩子（惊讶/好奇/想吐槽）
- 允许用感叹号、问号
- 人名/数字前置引流

### Step 3: 设计开头

前2句必须抓人，方法：
- 一句话定调（不超过20字）
- 或口语感叹切入
- 绝不用「近日，XX公司发布了...」这种公文腔

### Step 4: 组织正文

- 段落控制在1-3句
- 每300字插入一个情绪点（吐槽/感叹/调侃）
- 技术段落前后用口语句缓冲
- 大量使用加粗标记重点

### Step 5: 收尾

- 正文自然收束（展望/判断/调侃）
- 可选：加One More Thing
- 固定互动三连格式

## 关键风格信号

> 「没错，大洗牌之后，xAI紧锣密鼓地发上重磅新品了——」——典型的口语铺垫+破折号悬念

> 「不是？？从收到Prompt到交付，完全没停，一口气跑了整整28分钟。」——典型的情绪反应式叙事

> 「8位联创，再加少量创始团队，RSI总人数不超过30人」→ 紧接引用原话 → 「我们将尽可能保持团队小而精」——典型的事实+引用节奏

> 「也是很「拟人」了。（doge）」——典型的调侃收束
